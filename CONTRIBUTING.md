Thank you for your interest in contributing to this project!

Simple guidelines to get your changes accepted quickly:

- Fork the repository and create a feature branch (branch names like `feat/xxx`, `fix/xxx`).
- Keep changes focused and small; open separate PRs for unrelated changes.
- Run the project locally and ensure there are no obvious errors before opening a PR.
- Follow the existing code style. If you change formatting, try to keep it consistent.
- Add short, clear descriptions to your PRs explaining why the change is needed.

Automated checks

- This project uses ESLint. Please run `npm run lint` and fix issues before submitting a PR.
